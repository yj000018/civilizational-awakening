# Y-OS
## Posizionamento in una riga
Y-OS è un sistema operativo per la cognizione.
## Domanda fondamentale
In che modo gli esseri umani possono mantenere continuità, memoria, contesto e autonomia nei sistemi di IA?
## Tesi
L'era dell'IA non richiede solo modelli migliori. Richiede un'infrastruttura cognitiva migliore.
Mentre le persone lavorano tra ChatGPT, Claude, Manus, Cursor, browser, note, automazioni, file, riunioni e agenti, il contesto va perso. La memoria è parziale. Le decisioni scompaiono nelle conversazioni. Gli strumenti non sanno ciò che sanno gli altri strumenti.
Y-OS esplora il livello mancante: un sistema operativo cognitivo personale che cattura, consolida, instrada e inietta il contesto tra modelli, agenti, automazioni e sistemi di conoscenza.
## Cosa coordina Y-OS
Y-OS non è una singola app. È un'architettura per orchestrare:
- memoria,
- conoscenza,
- agenti,
- automazioni,
- prompt,
- flussi di lavoro,
- pacchetti di contesto,
- instradamento dei modelli,
- instradamento delle azioni,
- e conoscenza canonica.
## Architettura di base
L'attuale architettura di Y-OS utilizza un modello a livelli:
```plain text
Capture
↓
Recall
↓
Consolidate
↓
Canonicalize
↓
Route
↓
Inject
↓
Govern
```
In termini pratici:
```plain text
Recall captures.
mem0 serves.
Manus consolidates.
Obsidian remembers.
Git protects.
Agents receive context packs.
```
Per l'attuale fase di staging, Notion viene temporaneamente utilizzato come magazzino Markdown strutturato prima della migrazione verso Obsidian / Y-World.
## CRT e ART
Y-OS include due livelli di instradamento:
- **CRT — Cognitive Routing Table**: sceglie il modello o il sistema di ragionamento giusto per l'attività.
- **ART — Agent Routing Table**: sceglie l'agente, lo strumento o l'ambiente di esecuzione giusto.

L'attuale CRT v0 è manuale e semplice:
```plain text
GPT     = architecture, final specs, QA, code review
Claude  = review, simplification, human tone, long-form polish
Manus   = build, repo, backend, integration, deployment
Lovable = fast UI exploration
Gemini  = multimodal and large-context tasks
Image model = image generation and editing
```
Quando la fedeltà al contesto è fondamentale, GPT genera la prima bozza canonica. Claude poi la rivede e la semplifica. Questo impedisce a Claude di reinterpretare l'architettura di base troppo presto.
## Lineaggio intellettuale
Y-OS appartiene a un lignaggio di simbiosi uomo-computer e di divenire tecnologico. Douglas Engelbart è un riferimento chiave per l'aumento dell'intelletto umano. Kevin Kelly è un riferimento per comprendere la tecnologia come un ecosistema in evoluzione.
## Perché è importante
Senza un sistema operativo cognitivo, l'IA diventa una raccolta di strumenti potenti ma scollegati tra loro.
Y-OS si chiede come una persona possa preservare la continuità tra questi elementi: ciò che sa, ciò che ha deciso, ciò che sta costruendo, ciò che conta, ciò che dovrebbe essere ricordato, ciò che dovrebbe essere dimenticato e quale contesto ogni agente necessita in questo momento.
